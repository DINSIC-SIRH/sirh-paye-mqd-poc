STIFSCRUM-584 - En tant que RELAIS, je veux émettre des informations GetMultipleStopMonitoring 2.2 en mode requête

1 - Créer un producteur/diffuseur SNCF en 2.2
2 - Insérer le jeu de données correspondant au test voulu dans la base (IDENTIQUE STOP MONITORING)
3 - Envoyer la requete GetMultipleStopMonitoring (VOIR DEV_STORY_583/soapui) via SOAP UI sur le composant Emission
	


La balise "StopMonitoringFIlter" contient les même éléments que le StopMonitoring donc les tests doivent être identiques

<siri:StopMonitoringFIlter>
					<!-- IDENTIQUE STOP MONITORING -->

					<siri:MonitoringRef>STIF:StopPoint:Q:TEST1:</siri:MonitoringRef>


</siri:StopMonitoringFIlter>


Pour information : 
Le service GetMultipleStopMonitoring est traité exactement comme le StopMonitoring car utilisation des même méthodes
